
# Java 기반 Commerce 프로젝트

Java의 객체지향 원칙을 준수하여 카테고리별 상품 조회 시스템의 기초를 구현한 프로젝트입니다. 난이도상 필수 과제만 진행하였습니다.

---

### 필수 과제 내용

#### [STEP 1] 객체 지향 설계를 적용해 상품 관리 시스템을 프로그래밍 해보자.

#### [STEP 2] 객체 지향 설계를 적용해 순서 제어를 클래스로 관리하기

#### [STEP 3] 객체 지향 설계를 적용해 상품 카테고리와 고객 관리를 클래스 기반으로 관리하기

#### [STEP 4] 캡슐화 적용하기

---

### 프로젝트 구조

>- `Main.java` : 프로그램 시작. 전자제품, 의류 등의 데이터를 생성하고 시스템을 시작합니다.
>
>- `CommerceSystem.java` : 메뉴 출력 및 카테고리/상품 선택 로직을 담당합니다.
> 
>- `Product.java`, `Category.java` : 상품과 카테고리의 데이터를 정의하는 도메인 클래스입니다.
> 
>- `Customer.java` : 고객을 관리하는 도메인 클래스입니다.

### 주요 기능

>- 카테고리별 상품 조회 : 사용자가 선택한 카테고리에 맞는 상품 리스트를 화면에 출력합니다.
>
>
>- 뒤로가기 및 종료 : `0` 입력을 통해 상위 메뉴로 이동하거나 프로그램을 안전하게 종료할 수 있습니다.
>
>
>- 상품 상세 정보 제공 : 선택한 상품의 이름, 가격, 상세 설명 및 재고 수량을 한눈에 확인할 수 있습니다.

### 기술 스택

- Language : Java 17
- Concepts : OOP, Encapsulation, List Collections, Scanner Input

---

### 트러블 슈팅 및 고민 포인트

1. 메뉴 이동 시 무한 루프 및 흐름 제어
- 문제: 상세 페이지 확인 후 다시 '카테고리 선택' 화면으로 돌아가야 하는데, 반복문 제어가 잘못되어 원하지 않는 화면이 반복되는 문제가 있었습니다.
- 해결: 중첩 while 문 내에서 `displayProduct()` 호출 직후 `break`를 재배치하여, 하위 루프를 탈출하고 상위 루프인 카테고리 선택 화면으로 넘어가게끔 설계했습니다.



---

### 향후 개선 사항

1. 입력 예외 처리 보강 : 현재는 메뉴 선택 시 숫자만 처리 가능하나, 문자를 입력했을 때 발생하는 `InputMismatchException`과 같은 예외 처리 로직 추가할 예정

2. 장바구니 및 주문 시스템 구현 : `CartItem` 클래스를 만들어서 장바구니 관련 기능 다루기

3. 고객 관리 및 재고 연동 : `Customer` 객체를 활용한 회원 등급 시스템을 도입하고, 상품 구매 시 실제 Product 객체의 재고 수량이 실시간으로 차감되는 로직 설계
